# Monorepo Baseline

現時点のプロダクト前提と未確定事項をまとめる。

## 現在の目的

- Next.js と NestJS を同一リポジトリで運用する基本形を固める
- Web から API への疎通確認を常に行える状態を保つ
- 共有型、共有 UI、実行計画の置き場所を先に決めておく

## 現在のユーザー体験

- `apps/web` のトップページで API 疎通状態を確認できる
- `apps/api` は `/health` を公開し、JSON でステータスを返す

## 現在の契約

- API の既定ポートは `3001`
- Web の既定ポートは `3000`
- 共通契約は `packages/shared` に置く

## 未確定事項

- DB の採用有無と選定
- 認証方式
- API クライアント共有方式
- テスト戦略の拡張
- CI / CD
- デプロイ構成
