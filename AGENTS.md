# AGENTS.md

このファイルは、このリポジトリで作業する人やエージェント向けの作業ガイドです。

現時点では実装が未作成のため、内容は初期方針です。現在の前提構成は `Yarn Workspaces + Turborepo + Next.js + NestJS` です。

## リポジトリの前提

- このリポジトリは React.js と NestJS の練習用モノレポとして使う
- フロントエンド実装には Next.js を使う
- フロントエンドとバックエンドを同一リポジトリで管理する
- 共通コードはアプリ直下ではなく共有領域に切り出す
- ワークスペース管理には Yarn を使う
- タスク実行の統一には Turborepo を使う

## 想定ディレクトリ

```text
apps/
  web/    Next.js
  api/    NestJS
packages/
  shared/ 共通型やユーティリティ
  ui/     共通 UI が必要になった場合に利用
package.json
turbo.json
```

実際の構成が決まったら、この項目を実態に合わせて更新してください。

## 作業ルール

- 変更前に既存構成と設定ファイルを確認する
- 単一アプリだけで完結しない型や定数は共有領域に置く
- ワークスペース間の依存は Yarn Workspaces の定義に従って管理する
- 複数アプリにまたがる build / lint / test は Turborepo のタスク定義に合わせる
- 依存追加や構成変更をした場合は README を更新する
- 大きな方針変更を行う場合は、理由をドキュメントに残す

## 実装方針

- Frontend は Next.js を前提に進める
- Backend は NestJS を前提に進める
- API の入出力契約はフロントエンドとバックエンドでずれないよう管理する
- 共通型を持つ場合は `packages/shared` を優先して使う
- 命名、責務分割、ディレクトリ構成は一貫性を優先する

## コード品質

- Lint と Format の設定を早めに導入する
- テスト手段が決まったら最低限の実行手順を README に追記する
- タスク名は可能な限り `dev` / `build` / `lint` / `test` に揃える
- 破壊的変更を入れるときは、影響範囲を明記する

## ドキュメント更新ルール

- セットアップ手順が変わったら README を更新する
- ディレクトリ構成が変わったら AGENTS.md を更新する
- Turborepo のタスク構成を変えたら README か AGENTS.md に反映する
- 実装ルールが固まったら抽象的な表現を具体化する
